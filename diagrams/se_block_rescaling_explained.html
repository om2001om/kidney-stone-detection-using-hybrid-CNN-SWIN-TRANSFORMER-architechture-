⚙️ Channel Rescaling Mechanism

Understanding Broadcasting and Element-wise Multiplication in SE Blocks

🎯 The Big Picture

What is Rescaling?

Rescaling is the process of multiplying each channel in your feature map by its corresponding learned weight. It's called "rescaling" because you're literally changing the scale (magnitude) of each channel based on its importance.

Think of it like adjusting individual volume sliders on a mixing board - some channels get turned up (×0.95), others get turned down (×0.23).

1

You Have Features

A 3D tensor: 14×14×512

Think: 512 different "images" of size 14×14

Each "image" is a channel (feature map)

2

You Have Weights

A 1D vector: 512 values

Each value is between 0 and 1

One weight per channel (e.g., [0.95, 0.72, 0.34, ...])

3

You Multiply Them

Each 14×14 channel gets multiplied by its weight

Channel 0: multiply all 196 pixels by 0.95

Channel 1: multiply all 196 pixels by 0.72

And so on for all 512 channels...

📊 Visual Breakdown

🎯 Broadcasting: The Secret Sauce

What is Broadcasting?

Broadcasting is NumPy/PyTorch's way of handling operations on arrays of different shapes. Instead of manually repeating the smaller array to match the larger one, it does this automatically and efficiently.

1

Before Broadcasting

Original features: (8, 512, 14, 14)
Weights vector:  (8, 512)

Problem: Can't multiply directly!
Shapes don't match.
2

Reshape for Broadcasting

Reshape weights:
(8, 512) → (8, 512, 1, 1)

Add two dimensions of size 1
This tells PyTorch: "repeat this
across the missing dimensions"
3

After Broadcasting

Features:  (8, 512, 14, 14)
Weights:   (8, 512,  1,  1)
           ↓ broadcast to ↓
           (8, 512, 14, 14)

Now they match! Multiply element-wise.

Broadcasting in Action

# PyTorch Example

import torch

# Original features from CNN
X = torch.randn(8, 512, 14, 14)  # batch=8, channels=512, height=14, width=14

# Channel weights from SE block (after sigmoid)
s = torch.rand(8, 512)  # values between 0 and 1

# Reshape for broadcasting
s = s.view(8, 512, 1, 1)  # add two dimensions of size 1
# OR equivalently: s = s.unsqueeze(-1).unsqueeze(-1)

# Broadcast multiply (rescaling happens here!)
X_rescaled = s * X  # s broadcasts from (8,512,1,1) to (8,512,14,14)

# Result
print(X_rescaled.shape)  # torch.Size([8, 512, 14, 14]) - same as input!

# What actually happens:
# For each batch item (8 items):
#   For each channel (512 channels):
#     Take the scalar weight s[batch, channel]
#     Multiply ALL 14×14=196 values in that channel by this weight
#     Result: rescaled channel

🔢 Concrete Numerical Example

Let's trace through exactly what happens with real numbers for a single image (batch size = 1) with just 3 channels for simplicity:

Component Shape Sample Values
Input X (1, 3, 2, 2) 3 channels, each is a 2×2 matrix
Weights s (1, 3) [0.9, 0.5, 0.2]
Reshaped s (1, 3, 1, 1) Ready for broadcasting
Output X̃ (1, 3, 2, 2) Rescaled channels 
Channel 0 (weight = 0.9 - KEEP IT!):
Input:          [[0.8, 0.4],      ×0.9  →     Output:         [[0.72, 0.36],
                 [0.6, 0.9]]                                    [0.54, 0.81]]

Channel 1 (weight = 0.5 - NEUTRAL):
Input:          [[0.7, 0.3],      ×0.5  →     Output:         [[0.35, 0.15],
                 [0.5, 0.8]]                                    [0.25, 0.40]]

Channel 2 (weight = 0.2 - SUPPRESS IT!):
Input:          [[0.9, 0.2],      ×0.2  →     Output:         [[0.18, 0.04],
                 [0.4, 0.7]]                                    [0.08, 0.14]]

Interpretation:
• Channel 0: High weight (0.9) means it's important - keeps 90% of signal
• Channel 1: Medium weight (0.5) means it's somewhat useful - keeps 50%
• Channel 2: Low weight (0.2) means it's mostly noise - only keeps 20%

💡 Why This Works So Well

Spatial consistency: If a channel is important, it's important across the entire spatial extent. The SE block doesn't need to learn different weights for different spatial locations.

Efficiency: Only 512 weights needed instead of 512 × 14 × 14 = 100,352 weights!

Generalization: By learning global channel importance, the model generalizes better to different spatial resolutions.

💻 Implementation: Three Ways to Rescale

Method 1: view() - Most Common

b, c = x.size(0), x.size(1)
s = s.view(b, c, 1, 1)
out = x * s.expand_as(x)

Reshapes the weight tensor and uses expand_as to broadcast.

Method 2: unsqueeze() - Most Readable

s = s.unsqueeze(-1).unsqueeze(-1)
out = x * s

Adds dimensions explicitly. PyTorch broadcasts automatically.

Method 3: Indexing - Most Explicit

s = s[:, :, None, None]
out = x * s

Uses None (same as np.newaxis) to add dimensions.

Complete SE Block with Rescaling

class SEBlock(nn.Module):
    def __init__(self, channel, reduction=16):
        super().__init__()
        self.avg_pool = nn.AdaptiveAvgPool2d(1)
        self.fc = nn.Sequential(
            nn.Linear(channel, channel // reduction, bias=False),
            nn.ReLU(inplace=True),
            nn.Linear(channel // reduction, channel, bias=False),
            nn.Sigmoid()
        )
    
    def forward(self, x):
        # x shape: (batch, channels, height, width)
        b, c, h, w = x.size()
        
        # 1. Squeeze: Global Average Pooling
        y = self.avg_pool(x)  # (b, c, h, w) → (b, c, 1, 1)
        y = y.view(b, c)      # (b, c, 1, 1) → (b, c)
        
        # 2. Excitation: Learn channel weights
        y = self.fc(y)        # (b, c) → (b, c/16) → (b, c)
                              # sigmoid ensures values in (0,1)
        
        # 3. RESCALING: Reshape and broadcast multiply
        y = y.view(b, c, 1, 1)  # (b, c) → (b, c, 1, 1)
        
        # THIS IS WHERE THE MAGIC HAPPENS!
        # Broadcasting multiplies each channel by its weight
        # y gets broadcast from (b, c, 1, 1) to (b, c, h, w)
        rescaled = x * y.expand_as(x)  # Element-wise multiplication
        
        # OR equivalently (PyTorch handles broadcasting):
        # rescaled = x * y
        
        return rescaled

# Example usage
se = SEBlock(channel=512, reduction=16)
x = torch.randn(8, 512, 14, 14)
output = se(x)

print(f"Input shape:  {x.shape}")      # (8, 512, 14, 14)
print(f"Output shape: {output.shape}")  # (8, 512, 14, 14) - SAME!

📋 Quick Reference: The Rescaling Process

Step-by-Step Checklist

Step Operation Input Shape Output Shape What It Does
0 Input Features (B, C, H, W) (B, C, H, W) Original CNN features
1 Global Avg Pool (B, C, H, W) (B, C, 1, 1) Compress spatial dims
2 Flatten (B, C, 1, 1) (B, C) Remove spatial dims
3 FC1 + ReLU (B, C) (B, C/r) Compress to bottleneck
4 FC2 + Sigmoid (B, C/r) (B, C) Expand & generate weights
5 Reshape (B, C) (B, C, 1, 1) Prepare for broadcasting
6 Rescale (B,C,H,W) × (B,C,1,1) (B, C, H, W) Multiply each channel
7 Output (B, C, H, W) (B, C, H, W) Recalibrated features

🎯 The Bottom Line

Rescaling means multiplying each channel in your feature map by a learned weight (between 0 and 1).

Broadcasting is how PyTorch efficiently applies one weight to all 196 spatial locations in a channel.

Result: Important channels (high weights) keep their signal strength. Unimportant channels (low weights) get suppressed.

It's like having an intelligent, learned filter that adapts to each image, deciding which feature detectors to trust and which to ignore!
