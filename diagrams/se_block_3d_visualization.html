🧠 Fully Connected Layers: Deep Dive

What is a Fully Connected (FC) Layer?

A fully connected layer, also called a dense layer, connects every input neuron to every output neuron. If you have N inputs and M outputs, you need N×M weights plus M biases.

In the SE block, FC layers learn which channels should work together and which are independent.

🔢 FC Layer 1: Dimensionality Reduction

Purpose: Create an information bottleneck

Input: 512 channel descriptors

Output: 32 compressed features (r=16)

Parameters: 512 × 32 = 16,384 weights

y = W₁ · z
where W₁ ∈ ℝ^(32×512), z ∈ ℝ^512
Result: y ∈ ℝ^32

Why reduce? Forces the network to learn compact, essential channel relationships rather than memorizing patterns.

⚡ ReLU Activation: Non-linearity

Purpose: Add non-linear modeling capacity

Function: f(x) = max(0, x)

Effect: Zeroes out negative values

ReLU(x) = {
  x, if x > 0
  0, if x ≤ 0
}

Why ReLU? Enables learning complex, non-linear relationships between channels. Without it, FC1→FC2 would collapse to a single linear transformation.

📈 FC Layer 2: Expansion

Purpose: Map back to channel space

Input: 32 compressed features

Output: 512 channel weights

Parameters: 32 × 512 = 16,384 weights

s = W₂ · h
where W₂ ∈ ℝ^(512×32), h ∈ ℝ^32
Result: s ∈ ℝ^512

Why expand? Each of the 512 output values will become the importance weight for one channel.

🎯 Sigmoid Activation: Gating

Purpose: Convert to valid weights

Function: σ(x) = 1/(1+e^(-x))

Range: (0, 1) - perfect for weights

σ(x) ∈ (0, 1)
x = -∞ → σ(x) = 0 (suppress)
x = 0 → σ(x) = 0.5 (neutral)
x = +∞ → σ(x) = 1 (emphasize)

Why Sigmoid? Ensures all weights are between 0 and 1, allowing smooth channel recalibration without negative values.

🔬 The Power of the Bottleneck (r=16)

Without bottleneck: Direct 512→512 transformation = 262,144 parameters (prone to overfitting)

With bottleneck: 512→32→512 = 32,768 parameters (94% fewer!)

Benefit: The compression forces learning of generalizable channel relationships, not just memorization. The 32-dimensional space must encode the most important inter-channel dependencies.

→

→

// Example with actual numbers

Step 1: Squeeze
Input X: shape (batch=8, channels=512, height=14, width=14)
Global Average Pool: sum over H×W, divide by H×W
z = (1/196) × Σ(i=1 to 14)Σ(j=1 to 14) X[:, :, i, j]
Result z: shape (8, 512) - one global value per channel

Step 2: FC Layer 1
Input z: (8, 512)
Weight matrix W₁: (32, 512) 
Computation: y = z @ W₁.T = (8, 512) @ (512, 32) = (8, 32)
Result y: (8, 32) - compressed representation

Step 3: ReLU
Input y: (8, 32) with values like [-2.3, 0.5, 1.2, -0.8, ...]
ReLU(y): (8, 32) with values like [0, 0.5, 1.2, 0, ...]
Negative values → 0, positive values → unchanged

Step 4: FC Layer 2
Input h: (8, 32) - ReLU output
Weight matrix W₂: (512, 32)
Computation: s_logits = h @ W₂.T = (8, 32) @ (32, 512) = (8, 512)
Result s_logits: (8, 512) - back to channel space (unbounded values)

Step 5: Sigmoid
Input s_logits: (8, 512) with values like [-1.5, 2.3, 0.1, ...]
Sigmoid(x) = 1/(1+e^(-x))
s = σ(s_logits) gives values like [0.18, 0.91, 0.53, ...]
Result s: (8, 512) - all values in (0,1), these are channel weights!

Step 6: Scale (Broadcast Multiply)
Original features X: (8, 512, 14, 14)
Weights s: (8, 512) → reshaped to (8, 512, 1, 1) for broadcasting
X̃ = s × X = (8, 512, 1, 1) ⊙ (8, 512, 14, 14) = (8, 512, 14, 14)
Each channel scaled by its learned weight!

Example channel weights:
Channel 0: weight=0.95 → important, keep almost all signal
Channel 1: weight=0.23 → less important, suppress to 23%
Channel 511: weight=0.87 → quite important, keep 87%

🎨 Intuitive Understanding

🎭 Real-World Analogy

Imagine you're a radiologist looking at 512 different "views" of a kidney CT scan:

• View 1: Edge detection

• View 2: Texture patterns

• View 3: Density gradients

• ... 509 more views ...

SE Block's job: Learn which views are most important for detecting kidney stones. Maybe edges (0.95) and density (0.91) are crucial, but some random texture pattern (0.12) is noise.

🎛️ Mixing Console Metaphor

Think of the SE block as an audio mixing console with 512 channels:

Squeeze: Listen to each channel's overall volume (global avg pool)

Excitation: Decide which channels to boost/cut based on the song (FC layers learn the pattern)

Scale: Actually adjust the volumes (multiply by weights)

🔍 Why This Works

Problem: Not all 512 channels are equally useful for every image

Solution: Let the network decide dynamically which channels matter

Result: The same SE block behaves differently for different inputs - it "pays attention" to relevant features!

⚡ Computational Efficiency

Additional parameters: ~33K (only 0.1% of typical ResNet)

Additional computation: ~0.01% of total FLOPs

Performance gain: 2-5% accuracy improvement

Conclusion: Massive bang for buck! Tiny overhead, significant gains.
